torchtrain¶

torchtrain is a functional PyTorch neural network training library which provides high level building blocks and integrations instead of trying to do everything for you under the hood (most of current approaches, e.g. pytorch-lightning, skorch).

Using unified approach (single ** operator) across metrics, callbacks and others all users have to do is inherit from specific object (usually torchtrain.Operation) and implement desired forward or use provided building blocks by us.

Integrations¶

Integration are currently WIP, please file any issues you find along the way

Following are currently available:

  • comet-ml - module torchtrain.callbacks.comet, ‘Self-hosted and cloud-based meta machine learning platform allowing data scientists and teams to track, compare, explain and optimize experiments and models.’

  • neptune.ai - module torchtrain.callbacks.neptune, ‘The most lightweight experiment management tool that fits any workflow’

  • tensorboard - module torchtrain.callbacks.tensorboard, ‘Visualization and tooling needed for machine learning experimentation’

  • horovod - module torchtrain.accelerators.horovod and torchtrain.accelerators.Horovod, ‘Simple distributed training framework for TensorFlow, Keras, PyTorch, and Apache MXNet.’

All of those can be installed via extras, see below.

Installation¶

Following installation methods are available:

pip:¶

To install latest release:

pip install --user torchtrain

To install nightly version:

pip install --user torchtrain-nightly

torchtrain integrations come with extra. Simply run:

pip install --user torchtrain[neptune, tensorboard]

To install necessary packages for additional torchtrain.callbacks modules. Available extras:

  • [all] - install all extras in one go

  • [callbacks] - third party callbacks: [neptune, comet, tensorboard]

  • [accelerators] - third party accelerators: [horovod]

  • [horovod] - HOROVOD_CUDA_HOME=/path/to/cuda may be required during pip install

  • [neptune]

  • [comet]

  • [tensorboard]
